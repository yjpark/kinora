# Task: Migrate errors to thiserror (library) + rootcause (CLI)

**Goal.** Pairing thiserror (library, pure-typed) with rootcause (CLI,
report/render) is the 80% win with minimal library impact. The library errors
already implement `std::error::Error` manually and flatten inner errors into
Display strings; thiserror's `#[from]` gives proper `source()` chains for free.

**Done when:** all 15 library error enums derive `thiserror::Error` with
Display output byte-identical to today, the CLI wraps errors in `rootcause::Report`
with per-command context and shows a chained cause ≥1 level deep on a compound
error, `RUST_LOG`/`KINORA_TRACE` still work, tests pass, zero warnings.

## Outcome
- Commit 1: replaced hand-rolled `Error` + `Display` for 15 library enums
  (AssignError, CompactError, ConfigError, EventError, HashParseError, InitError,
  KinographError, StoreKinoError, LedgerError, NamespaceError, RenderError,
  ResolveError, RootError, StoreError, ValidationError) with
  `#[derive(thiserror::Error)]`. Dropped ~420 lines of boilerplate; Display
  byte-identical; `ResolveError::MultipleHeads` and `CompactError::AmbiguousAssign`
  field shapes preserved for CLI pattern-matching.
- Commit 2: migrated `kinora-cli::common::CliError` to thiserror (17 variants);
  added `rootcause` + `thiserror` deps; `report_err(command, e)` wraps via
  `rootcause::Report::new_sendsync` with `.context("`kinora {command}` failed")` and
  prints `{report}` (tree layout). All five dispatches route through it; the
  `MultipleHeads` fork report and `AmbiguousAssign` D2 hint are preserved.
- Chain verified with a corrupt `config.styx`: `kinora commit` prints a two-node
  tree (context + underlying `ConfigError::Parse`). 301 lib + 81 CLI tests pass,
  zero warnings.
