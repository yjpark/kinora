# Plan: Styxl writer + `kinora reformat` command

**Task:** kino://e76a42f7ead8dcf15967b50b04e0137ef982729a8b2fc8102e9bf753952b32f7

The original plan was a `.multiline()` writer switch; that path proved blocked by a
facet-styx short-circuit, so it pivoted to the styxl format. Phases below are the
revised, shipped sequence.

## Phase 0 — `.multiline()` writer switch *(abandoned)*
Attempted a `kinora::styx::write` helper wrapping
`to_string_with_options(.., multiline())` across config.rs/root.rs/kinograph.rs.

> **Done (as a dead end, reverted).** Empirically failed: facet-styx 3.0.2 forces
> non-root structs/seqs to `inline_start: true`, so `ForceStyle::Multiline` is
> short-circuited and only top-level fields break onto lines. `format_source` post-
> pass also ignores `force_style`. Reverted to keep the tree clean; pivoted to
> styxl (see kino://b21e570f46d930684d4d0fe3a0bd6f1b454998c58fe5b1ee2c6612bf60e62888).

## Phase 1 — styxl serialization for Kinograph + RootKinograph *(done)*
`to_styxl`/`parse_styxl`/`is_styxl`; switch kinograph-kind writes to emit `.styxl`;
update kind→extension mapping.

> **Done.** Round-trip, empty-entries, escaped-strings/long-metadata, and per-line
> parse-independence tests pass. `find_blob_path` strips extensions so lookups
> survive the transition.

## Phase 2 — `kinora reformat` library + CLI *(done)*
Walk reachable kinograph kinos, rewrite `.styx`-wrapped → styxl, stage new-version
events; root kinographs rewritten directly via `store_kino(kind:"root", ...)`.

> commit-makes-it-head). Review fixes: NoHead variant, UTF-8 guard, hoisted ledger
