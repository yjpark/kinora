# Cache path & config: repo-url identity, `~/.cache/kinora/<shorthash>-<name>/`

Settles RFC-0003 Q7 for the kinora bootstrap: project identity, the render cache path, and the config file.

## The decision
- **Chosen: repo URL is the canonical project identity.** Multiple clones share one cache; different projects with the same name don't collide.
- **Chosen: cache path `~/.cache/kinora/<shorthash>-<name>/`** — `shorthash` = first 8 hex of `BLAKE3(normalized-repo-url)`, `name` = last path segment of the normalized URL sanitized to `[a-z0-9_-]`. Nix-style `<hash>-<name>` is readable but unambiguous.
- **Chosen: URL normalization** strips scheme, user-info, `.git`, and trailing slash; converts SSH `:` → `/`; lowercases host (path preserved). So `https://` and `git@` forms of the same repo collide as intended.
- **Chosen: `.kinora/config.styx` with a single required `repo-url` field**; other fields (author defaults, render options) optional. `kinora init` auto-fills `repo-url` from `git remote get-url origin`, else errors and prompts.
- **Rejected: directory-name or path-based project identity, and multiple required config fields.** Path identity collides across clones; extra required fields complicate setup.
- **Not a dead end:** optional config fields can grow without breaking the single-required-field contract.
