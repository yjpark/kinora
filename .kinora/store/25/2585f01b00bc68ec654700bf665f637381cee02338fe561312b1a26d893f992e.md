# Task: Reformat synthesizes nested composition entries from pins

**Goal.** Close wcpp's migration-debt gap for nested kinograph entries — those
referenced only via another kinograph's composition list (not themselves root
entries) whose store event was drained — by using the composition entry's `pin` as
a version hint to synthesize a stub and reformat the inner kinograph on the same
pass.

**Realizes:**
- kino://5b3415a320224957e71ea61f63683c56b498f64abb1929e9bf8c512206b898e8 — Drain archived events from staging for MaxAge.
- kino://b21e570f46d930684d4d0fe3a0bd6f1b454998c58fe5b1ee2c6612bf60e62888 — Kinograph kinos use the styxl one-entry-per-line format.

**Done when:** reformat produces new versions for both an outer root-entry
kinograph and a nested, drained-only inner kinograph it pins; a nested entry
without a pin (and no staged/root event) is still silently skipped; a pin pointing
at non-kinograph content does not synthesize a kinograph stub; no regression; zero
warnings.

## Outcome
Shipped in 3 commits (522 tests). `to_visit` changed from `Vec<String>` to
`Vec<(String, Option<String>)>` carrying a version hint — root-entry seed pushes
`(id, Some(entry.version))`, composition recursion propagates
`entry.pin_opt()`. A synth-from-hint block reads the hint hash, verifies it parses
as `Kinograph`, and injects a kinograph-kind stub into `events_by_id`; NotFound is a
silent skip, other store errors propagate. 3 tests added. Low priority because
reformat is idempotent — a subsequent commit cycle re-surfaces missed entries — and
the data still lives in the archive. No debug log added for the no-pin silent skip
(no observability need surfaced).
