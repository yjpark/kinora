# Task: Group rendered pages by owning root (not legacy branch label)

**Goal.** Under the hot-ledger layout new events don't belong to any legacy
per-lineage ledger file, yet render still grouped pages under a branch label derived
from the old `.kinora/ledger/<lineage>.jsonl` naming (the misleading `7a155b58`
label applied to every page). Decide what grouping means and fix it.

**Done when:** rendered pages are grouped by owning root kinograph name; kinos not
in any root land in an `unreferenced/` bucket; tests cover pure-staged, single
`main` root, and mixed repos; zero warnings.

## Outcome
- Decision: group by **owning root** — under the xi21 ownership model every kino has
  exactly one owning root; today only `main` exists so it's a single bucket, but the
  structure is future-proof for multi-root without render-layer rework. Pre-commit
  kinos group under `unreferenced/`.
- Library (`crates/kinora/src/render.rs`): `render_for_branch(resolver, branch)` ->
  `render(resolver, labels: &HashMap<String,String>, default_label: &str)`;
  `RenderedPage.branch` -> `.group`; helpers renamed; source marker reworded; skip
  `kind == "root"` kinos.
- CLI (`crates/kinora-cli/src/render.rs`): removed `current_branch_label`; added
  `build_owners_map(kin_root)` scanning `.kinora/roots/`, loading each root blob via
  `ContentStore`, recording every entry id under its root name (sorted for
  determinism; tmp/non-file entries skipped); added `CliError::{Store, Root}`.
- Tests: label-from-map, default fallback, root-kind skip, owners-map cases, and
  three end-to-end renders (pure-staged `unreferenced/`, compacted `main`, mixed).
  231 + 54 tests pass, zero warnings (commits 837e921 / d45aa6e / 19d850a).
