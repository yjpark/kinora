# Stencil: a kinora-native, language-agnostic crate-API preprocessor

RFC-0004 settles how a crate's public API — held as kinora kinos — is rendered
into read-only sections of source files while editable regions are preserved.
This spec captures the overarching stencil architecture; the marker protocol and
spec content model are split into their own specs.

## The decision
- **Chosen: a silp-inspired, kinora-native, in-place preprocessor over a
  separate generated-output pipeline.** Stencil is a new *renderer* with a
  source-file target. Inputs are kinora kinos (`kudo::api-spec` /
  `kudo::api-kinograph`); design is language-agnostic; Rust is the bootstrap
  target. Packaging mirrors the kinora/kinora-cli split: `crates/stencil` (lib,
  depends on the `kinora` crate) + `crates/stencil-cli` (bin `stencil`).
- **Chosen: in-place, agent-placed slots over a fully tool-owned file layout.**
  The agent drops a `// stencil:slot <entry-name>` marker wherever it wants the
  item; stencil fills/refreshes only its own read-only block at that slot and
  never touches anything else. Maximum layout freedom; honors "stencil is the
  sole author of read-only sections".
- **Chosen: kinograph-bound sync over file-local membership.** "API as
  kinograph" makes the kinograph the source of truth for membership + pinning. A
  file declares its binding once (`// stencil:kinograph <name|id>`); `stencil
  sync` loads that api-kinograph (reusing kinora's `Kinograph` — the
  api-kinograph blob is the same `{id,name?,pin?,note?}` styxl entry shape),
  matches each slot to an entry by name, resolves it (entry `pin` if set, else
  head, via kinora `Resolver`), splits the spec kino, and writes the read-only
  block. Pinning rides the existing `Entry::pin` — no new mechanism.
- **Rejected: requiring kinora-crate changes.** `kudo::api-spec` +
  `kudo::api-kinograph` already pass `validate_kind` (namespaced);
  api-kinograph content parses via `kinora::kinograph::Kinograph`; resolution
  uses `kinora::resolve::Resolver`. Stencil adds only a thin kind-scoped name
  lookup over `Resolver::identities()`. No kinora changes required.
- **Not a dead end:** the language target is pluggable (see the LanguageTarget
  decision), the marker spelling is refinable, and spec-versioning / drift
  enforcement / extra targets are deferred behind clean seams.

## What this built
- The first slice: the render/sync engine plus a day-one dogfood — stencil's own
  public API defined as kinographs and its source rebuilt from them. Verified by
  `stencil sync` filling every read-only block and a second sync reporting 0
  changed (self-hosting), with the workspace green.
