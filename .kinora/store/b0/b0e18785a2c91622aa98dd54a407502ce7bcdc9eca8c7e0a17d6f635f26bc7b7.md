# Task: Repack drains already-archived staged events

**Goal.** Add a post-commit, pre-clone drain pass in `repack_repo` that drops
staged events whose hashes already appear in a `commit-archive` kino, closing
wcpp's migration-debt gap for repos that can't self-clean via a no-op commit.

**Realizes:**
- kino://5b3415a320224957e71ea61f63683c56b498f64abb1929e9bf8c512206b898e8 — Drain archived events from staging for MaxAge.
- kino://f1a123d9f07b03c46af0b372fba2303a21889c57ccb6450e9e202292ba6b6420 — `kinora repack` composes commit + clone-to-temp + atomic swap.

**Done when:** a pre-wcpp staged event already in an archive is drained by repack;
an unarchived staged event stays put; a KeepLastN source root's archived events are
NOT drained; no regression of repack/commit tests; zero warnings.

## Outcome
Bugfix shipped in 3 commits (519 tests). New public
`drain_archived_orphans(kinora_root) -> Result<usize, CommitError>` in commit.rs
walks the commits-root kinograph, picks `commit-archive` entries, parses the source
root from the `<source_root>-commit-archive` metadata name, and for `Never | MaxAge`
source roots reads each archive blob, collects event hashes, and drops them from
staging (tolerates missing commits pointer and reaped blobs, both `Ok(0)`).
`repack_repo` calls it between `commit_all` and `clone_repo`; new
`RepackReport.orphan_events_drained` surfaced in CLI output. 6 unit + 1 e2e test.
