# Task: Kinograph composition format implementation

**Goal.** Implement the kinograph kind: parse/validate/round-trip styx `entries[]`, resolve entry names to ids at store time, and render a kinograph by inlining resolved entries in order.

**Realizes:**
- [[kinora-zboo:spec]] — kinograph composition format.
- [[kinora-6zxd:task]] — reuses resolver name→id semantics.

**Done when:** styx kinographs parse with `entries[]`; entry shape `{id, name?, pin?, note?}` is validated; store-time name→id resolution errors on ambiguous/missing names; pinned refs resolve to a specific content hash; raw files stay human-readable; updates append new ledger events through the kind-agnostic store path; and the renderer concatenates resolved entries in order with per-entry notes as blockquotes. Verified by library + CLI tests.

## Outcome
- **Library** (`kinograph.rs`) — `Entry { id, name, pin, note }` as facet-styx records; all optional-feeling fields are `String` with `#[facet(default)]` and an empty-means-absent convention (facet-styx can't round-trip `Option<String>`), exposed via `name_opt()/pin_opt()/note_opt()`. `Kinograph::{parse, parse_str, to_styx}` round-trip the inline form. `resolve_names(&Resolver)` resolves any entry whose `id` isn't valid 64-hex via its `name` (or the id slot treated as a name), filling the empty `name`; errors on ambiguous/missing. `render(&Resolver)` walks entries, fetches by pin (`resolve_at_version`) or head (`resolve_by_id`), emits `note` as `> `-prefixed lines, joins with blank lines. `KinographError`: Parse, Serialize, InvalidEntry, Resolve, Utf8, AmbiguousName, NameNotFound.
- **CLI** (`store.rs`) — when `kind == "kinograph"`, stored bytes are the re-serialized kinograph after `resolve_names` (authoritative by id, name hints preserved). Error surface extended with `CliError::{Resolve, Kinograph}`.
- **Format decisions** — pin semantics align with `event.hash = BLAKE3(content)`; current-name-drift warning deferred (hint preserved for a future lint).

10 library tests + 4 CLI tests, zero warnings.
