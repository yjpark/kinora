## Outcome
- **Library** — `author::resolve_author_from_git()` reads `user.name` from the repo's merged git config via gix (None for non-git/unset). `kino::store_kino()` orchestrates write-blob (dedup) → birth-vs-version decision (`--parents` without `--id` rejected as `ParentsWithoutId`) → build envelope → `validate_event_shape` + `validate_parents_exist` → mint-or-append by HEAD presence.
- **CLI** — figue-derived `Cli`/`Command::Store` (positional `kind`+`path`, required `--provenance`, optional `--name`/`--id`/`--parents`/`--author`, `--draft`, repeatable `-m key=value`; flattens `FigueBuiltins` for `--help`/`--version`/`--completions`). `common.rs` walks up to find `.kinora/`, parses `KEY=VALUE` (values may contain `=`), splits comma parents. `store.rs` reads file/stdin, assembles metadata, resolves author (flag > git), stamps via jiff; rejects `--draft` + `-m draft=X` collisions (`ConflictingDraftFlag`). `main.rs` exits 0 for help/version/completions, 1 for failures; prints `stored kind=… id=… hash=… lineage=…`.
- **MVP lineage model** documented inline: one lineage per repo (HEAD-absent mints, present appends); per-branch/per-identity partitioning deferred — identity lives in `event.id`, not the lineage filename.
