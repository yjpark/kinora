# Task: Rename hot -> staged, compact -> commit

**Goal.** Replace the "hot"/"compact" jargon (carried from an earlier mental model)
with "staged"/"commit", which map directly onto git vocabulary users already carry.
Pure rename — no lifecycle change; staged events remain in place after commit.

**Done when:** `kinora commit` works identically to the old `kinora compact`, no
residual `hot`/`compact` identifier remains in library code, the workspace tests
pass, and there are zero warnings.

## Outcome
- Paths: `HOT_DIR` -> `STAGED_DIR`, `hot_dir()` -> `staged_dir()`,
  `hot_event_path()` -> `staged_event_path()`, literal `"hot"` -> `"staged"`.
- Module/files: `compact.rs` -> `commit.rs` in both `kinora` and `kinora-cli`.
- Types: `CompactError`/`CompactParams`/`CompactResult`/`CompactAllEntry`/
  `CompactRunArgs`/`CompactRunReport` -> `Commit*` (all error variants preserved).
- Functions: `run_compact`/`compact_all`/`compact_root`/`render_compact_entry`/
  `render_compact_error` -> `commit`/`commit_*`.
- CLI: subcommand `compact` -> `commit` (`Command::Compact` -> `Command::Commit`).
- Dogfood repo: `.kinora/hot/` -> `.kinora/staged/` via `git mv`.
- Hard rename, no transitional aliases (repo days old, no external users). 382 tests
  pass, zero warnings, independent review found no missed references.
