# Task: Machine-readable output for `kinora store`

**Goal.** `kinora store` prints `id=<hash>` (with `=`), easy to mis-parse and a
`set -e` footgun when scripting. Provide stable machine-readable output for store.

**Done when:** a global `--json` flag makes `kinora store` emit a single-line JSON
object that round-trips through a JSON parser with no `key=value` form.

## Outcome
- `crates/kinora-cli/src/cli.rs`: added global `Cli.json` flag.
- `crates/kinora-cli/src/store.rs`: `StoreJson` struct + `format_store_json`
  emitting `{kind,id,hash,event,new_event}` via `facet_json` with a safe hand-built
  fallback (all fields constrained: hex hashes, namespace-validated kind, bool).
- `crates/kinora-cli/src/main.rs`: store branch emits JSON when `--json` is set.
- `crates/kinora-cli/Cargo.toml`: added `facet-json`.
- Scope note: `--json` is honored by `store` only; assign/commit/resolve/render
  remain a documented follow-up.
- Tests: `format_store_json_emits_parseable_fields`,
  `format_store_json_reflects_idempotent_restore`. All kinora-cli tests pass, zero
  warnings.
