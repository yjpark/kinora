## Outcome
Shipped the full data layer plus library-level `init` across 7 focused commits (tests-first, zero warnings), 82 unit tests:
- **hash / paths** — `Hash` newtype (64-char hex, `shorthash`/`shard`), on-disk path builders.
- **value / namespace** — `Value`/`Metadata` with null-removes-field merge; reserved bare kinds/keys + `prefix::name` extension rule.
- **config** — facet-styx `Config`, MVP field `repo-url`.
- **store** — `ContentStore::write` (atomic tmp-rename, idempotent, sharded) + `read` (re-hashes, detects corruption).
- **event / ledger** — facet-derived envelope with JSONL round-trip and `is_birth()`; `mint_and_append` / `append_to_head` / HEAD tracking. Append-only enforced at the OS level (mint = `create_new(true)` → `LineageAlreadyExists`; append never creates → `LineageMissing`; `set_head` atomic via tmp-rename).
- **validate** — shape checks (namespace, hash parseability, birth/version id-hash rules, self-parent + duplicate-parent rejection) plus content-backed checks via `store.read()` (`ParentCorrupted`, `EventHashCorrupted`).
- **init** — refuses existing `.kinora/`; `resolve_repo_url_from_git()` via gix; `init_with_git_fallback()` prefers `--repo-url` else reads `origin`.

Branch-aware lineage minting deferred to the `store` command (HEAD-absent ⇒ mint, else append).
