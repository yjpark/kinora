# Task: Store filename appends a kind-derived extension

**Goal.** Make `ContentStore::write(kind, bytes)` write to `<hash>.<ext>` with the
extension derived from kind, and make reads resolve by hash stem so the extension
is advisory.

**Realizes:**
- [[kinora-wpup:spec]] — Store blobs carry a kind-derived extension; reads are extension-agnostic.

**Done when:** `write` emits `<hash[..2]>/<hash>.<ext>`; `read` resolves via stem
scan regardless of extension; legacy extensionless layouts still readable; tests
cover per-kind round-trip, glob read, same-content-different-kind dedup, and
namespaced-kind fallback; zero warnings.

## Outcome
Landed in 3 commits. `ext_for_kind` (namespace.rs) + `find_blob_path` (paths.rs)
added; `ContentStore::write` picks the extension, `read`/`exists` use stem
matching; legacy extensionless blobs still resolve; dedup unchanged (first writer
wins the extension). Review fix changed the tmp name from `<hash>.tmp` to
`.tmp-<hash>[.ext]` so a crashed partial write can't be mistaken for a real blob by
the stem scan. RFC-0003 `.md` re-import deferred (legacy blobs remain readable).
