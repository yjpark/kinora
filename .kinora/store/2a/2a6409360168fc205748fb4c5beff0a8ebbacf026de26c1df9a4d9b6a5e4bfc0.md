**Goal.** Add an event-kind discriminator to hot-ledger events, distinct from the content `kind`, so future kinds (`assign`, metadata-resolve) can coexist with store events — without breaking legacy store-event parsing.

## Outcome

Added `EVENT_KIND_STORE`, `Event::new_store(...)`, `is_store_event()`, and a `from_json_line` fallback triggered only on the "missing field `event_kind`" primary error. 4 new event tests + consumer filter tests; 303 workspace tests green.
