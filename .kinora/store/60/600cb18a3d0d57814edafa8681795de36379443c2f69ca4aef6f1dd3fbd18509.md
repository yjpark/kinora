# Task: Warn on uncommitted kino state at render time

**Goal.** `kinora render` reads git-committed `.kinora/`, not the working tree, so
un-`git commit`-ed kino activity is silently absent from the rendered site. Add a
kinora-side warning when the working tree differs from committed state to make the
store -> `kinora commit` -> `git commit` -> render footgun visible.

**Done when:** `kinora render` warns to stderr when the working-tree `.kinora/`
differs from the version committed at the current branch HEAD, and stays quiet when
they match or when there is nothing to compare.

## Outcome
- `crates/kinora-cli/src/render.rs`: added `RenderReport.uncommitted_kino_state`,
  `working_tree_has_uncommitted_kino_state()` (best-effort: extracts HEAD's
  `.kinora/` to a scratch dir and byte-compares to the working copy; returns false
  on any error, non-git repo, unborn HEAD, or no committed `.kinora/`; never blocks
  the render), plus `dirs_differ` / `collect_rel_files` recursive helpers.
- `crates/kinora-cli/src/main.rs`: prints the advisory warning to stderr.
- Tests: `dirs_differ` add/remove/change + recursion; render flags uncommitted
  state; quiet when committed; quiet for non-git repo. Full workspace green, zero
  warnings.
