# Root kinographs are self-contained: commit metadata lives in a header line

Settles where a root's commit metadata (lineage id, parents, ts, author, provenance) lives. After commit + repack, the old model left one `kind: root` store event per root in staging forever — durable commit metadata masquerading as pending work.

## The decision
- **Chosen: inline commit metadata as a `RootHeader` line at the top of the root styxl blob; write no separate root store event at all.** Root kinographs then self-contain their full commit chain. Line 1 is `{kind root, id, parents [...], ts, author, provenance}`; lines 2+ are the (unchanged) entries. Disambiguation is unambiguous: the header has `kind root` + `parents`, entries have `version` + `head_ts`. Genesis → `parents []`. After commit + repack, `.kinora/staged/` holds only pending content/assign events (empty after repack).
- **Rejected: keeping a `kind: root` store event in staging.** It can never be drained without breaking the version chain, makes repack awkward, and violates the "staged = new events, not yet promoted" mental model.
- **Chosen: hard cutover, no migration code.** The new format is the only root format; styx-wrapped `entries (…)` and header-less styxl no longer parse. Kinora is pre-1.0, so existing repos upgrade by `rm -rf .kinora/ && kinora init && <re-store>`. History before cutover is not preserved.
- **Chosen (lineage id, option A): genesis header `id` = hash of the entries-only bytes (excluding the header line)**, carried forward unchanged through every subsequent version. Mirrors kino identity. Rejected option B (derive deterministically from root name) — chosen A keys identity to genesis content. Parser treats id as informational (does not re-verify).
- **Not a dead end:** no extra header fields beyond id/parents/ts/author/provenance (YAGNI); merging `store_kino` and `commit_root_with_refs` into one path is deferred.

## What this built
- `RootHeader` struct + `RootKinograph::new_genesis`/`new_child`, header-first `to_styxl`/`parse_styxl`, `commit_root_with_refs` writing the blob + `roots/<name>` pointer directly (no staged event). `reformat.rs` Step 1 (legacy root rewrite) and the `PriorRootEventMissing` error both removed; `ledger`/`clone`/`repack` no longer special-case root events. Verified: header/genesis/multi-parent roundtrips, 3-version lineage-id stability, post-commit+repack `staged/` empty, `WrongHeaderKind` rejection. See [[kinora-et1t:task]]. Commits `4e80bf5`, `b64f6cf`, `7e79803`.
