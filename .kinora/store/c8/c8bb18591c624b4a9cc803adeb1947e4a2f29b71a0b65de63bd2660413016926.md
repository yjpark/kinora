# Task: Global `--repo-root` / `-C` flag

**Goal.** Every command starts with `find_repo_root(cwd)`, which walks up from cwd
for `.kinora/`. That is awkward for tempdir-driven tests, multi-repo scripts, and the
upcoming clone/repack commands. Add a git-style `-C <path>` / `--repo-root <path>`.

**Done when:** the flag works on all subcommands; with `--repo-root` given it is used
verbatim (erroring with `NotInKinoraRepo` if `.kinora/` isn't directly under it, no
walk-up); omitted, behavior is unchanged; zero warnings, all tests pass.

## Outcome
- `crates/kinora-cli/src/cli.rs`: `repo_root: Option<String>` with `-C` alias on the
  top-level `Cli` struct.
- `crates/kinora-cli/src/common.rs`: `resolve_repo_root(cwd, override_path)` —
  verbatim when override is `Some`, walks up via `find_repo_root` when `None`. Four
  unit tests cover both branches plus the "doesn't silently walk up from a subdir"
  edge case.
- `crates/kinora-cli/src/main.rs`: calls `resolve_repo_root` once in `run()` and
  shadows `cwd`.
- Manual verification: `--help` shows the flag; `-C /tmp/test` resolves to the
  tempdir; `-C /nonexistent` errors with `NotInKinoraRepo`.
- All 386 workspace tests pass, zero warnings.
