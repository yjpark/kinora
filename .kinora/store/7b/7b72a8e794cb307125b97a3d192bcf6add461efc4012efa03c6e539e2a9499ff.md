# Task: Implement inline-only kino rendering

**Goal.** Add an `inline=true` metadata signal so a kino can be composed as a
kinograph member without leaking out as its own standalone render page.

**Realizes:**
- [[kinora-qzfv:spec]] — Inline-only kinos via `inline=true` metadata.

**Done when:** a kino with `inline=true` is skipped from page emission but still
inlines into any kinograph that composes it; `inline=false`/other values still
render a page.

## Outcome
- `crates/kinora/src/namespace.rs`: `inline` registered as a reserved bare key.
- `crates/kinora/src/render.rs`: `render()` suppresses page emission only when head
  metadata is exactly `inline=true`; inlining via `Kinograph::render` is unchanged.
- Tests cover all three cases. Full workspace green, zero warnings.
