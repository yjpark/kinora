# Task: CLI `resolve` command

**Goal.** Given a kino name or id, return its current content — building the within-identity version DAG, detecting forks, and supporting branch-aware tiebreaks and specific-version lookup.

**Realizes:**
- [[kinora-5k13:task]] — data layer / ledger scan it sits on.
- [[kinora-fhw1:spec-3]] — version DAG, name-as-metadata lookup.
- [[kinora-fhw1:spec-4]] — fork detection, branch-aware resolution.

**Done when:** `resolve <id>` returns the current head's content; `resolve <name>` does metadata-name lookup and errors on ambiguity; fork detection traverses the within-identity DAG so a single head returns content and multiple heads refuse with an actionable report (heads listed, reconcile commands shown); branch-aware tiebreak prefers the head in the HEAD lineage; `--version HASH` returns a prior version; `--all-heads` lists all heads without erroring; unknown name/id gives a clear error. Verified by library + CLI tests.

## Outcome
- **Library** (`resolve.rs`) — `Identity` groups all events of one id across lineages; heads are precomputed as events not referenced as a parent by any same-identity event. `Resolver::load` reads all identities from `Ledger::read_all_lineages`. `resolve_by_id`, `resolve_by_name` (scans `metadata.name` on heads so renames take effect), `resolve_at_version`. Branch-aware tiebreaker: if multiple heads but exactly one lives in the HEAD lineage, it wins; otherwise `MultipleHeads` carries heads + lineage shorthashes. Content is re-verified via the hash-checking `ContentStore::read`.
- **CLI** (`resolve.rs`) — `run_resolve(cwd, args)` → `ResolveOutcome` (content or all-heads listing). `--version HASH` resolves identity first (by hash or name) then reads the prior version. `--all-heads` turns `MultipleHeads` into a clean listing. Fork report matches spec (`kino <name> (id: shorthash…) has N heads: … Reconcile via one of: merge / linearize / keep-both / detach`). Output contract: content → stdout, fork report → stderr + non-zero exit.
- **Design notes** — name lookup matches only the current head's metadata (renames take effect; old name → NotFound); `--version` by name resolves identity from the head first; `SHORTHASH_LEN` constant used throughout.
