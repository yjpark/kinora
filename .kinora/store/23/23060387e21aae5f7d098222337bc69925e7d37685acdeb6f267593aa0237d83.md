# Task: Multi-branch + worktree render enumeration

**Goal.** Extend `kinora render` to union committed `.kinora/` state across all local
branches and worktrees, rendering each as a top-level SUMMARY.md section.

**Plan:** [[kinora-ohwb:plan]]

**Done when:** render enumerates `refs/heads/*` + linked worktrees, emits one
top-level group per source, a kino committed only on branch B appears only under B,
and kinos on multiple branches render once per branch with a branch-cited source
marker.

## Outcome
- New `kinora::git_state` module reads committed `.kinora/` per branch/worktree via
  gix 0.81 (no working-copy dependency): `extract_subtree`, `list_local_branches`,
  `list_worktrees`.
- `kinora render` enumerates branches + linked worktrees, deduped by ref name;
  detached-HEAD worktrees keep a `worktree-*` label; pre-kinora branches skipped
  per-branch via `ExtractError::SubtreeAbsent`; pre-first-commit falls back to the
  working copy with legacy root-based grouping. `RenderReport.source_count` reports
  contributing sources.
- Decisions realized: Q1 per-branch rendering with duplicated pages (no cross-branch
  dedupe); Q2 `kino://<id>/` resolves within-branch only.
- Known limitations (follow-ups): `gix::open(repo_root)` doesn't walk up; worktree
  labels use directory basename.
- 11 render-focused kinora-cli tests + 8 git_state tests; 489 workspace tests pass,
  zero warnings (commits 1ae4128 / e5e0ddb / 1a3360f).
