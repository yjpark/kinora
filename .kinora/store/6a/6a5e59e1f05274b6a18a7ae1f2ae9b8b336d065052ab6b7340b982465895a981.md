# Task: compact consumes assigns + AmbiguousAssign + UnknownRoot

**Goal.** Teach `compact_root` to consume live `assign` events and enforce the two fail-loud errors the design requires (D2/D4). This is where exclusive ownership becomes real.

**Realizes:**
- [[kinora-xi21:spec-5]] — Root ownership is exclusive and structural; kinos move via assign events.

**Plan:** [[kinora-hxmw:plan]]

**Done when:** a single live `assign → Y` moves a kino into Y (and out of its prior root on the next compact of both); superseded assigns (transitively) are not live; unassigned kinos route to `inbox`; ≥2 live assigns raise `AmbiguousAssign` with candidate details; an assign to an undeclared root raises `UnknownRoot`; and the CLI renders both errors in the D2 format.

## Outcome
Resolved. `collect_live_assigns` computes the live graph by dropping any assign named in another assign's `supersedes` (transitive falls out from the all-at-once superseded set). `kino_target_root` returns inbox-default / single-target / `AmbiguousAssign` / `UnknownRoot`. `build_root` now takes `declared_roots: &BTreeSet<String>` and routes before `pick_head`. CLI `render_compact_entry`/`render_compact_error` emit the D2 multi-line ambiguous block (padded target column + copy-pasteable resolution hint) and the UnknownRoot line. Cross-root removal falls out free because root content is rebuilt from hot events. Bulk test migration to compact-to-inbox. 287 lib + 77 CLI tests. Commits `34b0f64`, `5f52f3c`, `11a2d8e`.
