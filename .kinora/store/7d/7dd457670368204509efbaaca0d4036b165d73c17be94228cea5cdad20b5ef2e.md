# Kino model: content-addressed identity + version DAG + latest-wins metadata

Settles RFC-0003 Q3 for the kinora bootstrap: the core kino data model, kinograph reference format, and namespacing. This is the foundational model the whole system is built on.

## The decision
- **Chosen: identity = BLAKE3 of the first content version (immutable); version = content hash; metadata = latest-wins per field.** Content-addressed identity matches Kinora's ethos; metadata as latest-wins gives a clean data/metadata storage line (git commit vs git notes analogy).
- **Chosen: name is a metadata field, not identity; no uniqueness enforced.** Names are hard to get right up front and should not be load-bearing.
- **Chosen: a version DAG where each version's `parents[]` may cross identities.** Cross-identity parents unify linear history, fork, detach (new id, cross-identity parent), and combine (new id, multiple cross-identity parents) without inventing new event kinds.
- **Chosen: kinograph format = styx with `entries[]`, each `{id, name?, pin?, note?}`** — `id` authoritative, `name` a non-authoritative hint, `pin` optionally freezes to a specific version.
- **Chosen: namespace convention — bare names are Kinora-reserved; extensions are `prefix::name`** — applied to metadata keys, ledger event `kind`, and kinograph entry kinds. Validation is strict-on-write (reject unknown bare names), permissive-on-read (preserve unknown namespaced names). Supports open-closed extension.
- **Rejected: enforced unique names / name-as-identity.** Couples identity to a mutable, contested field.
- **Not a dead end:** variants/detach are deferred to post-bootstrap; the schema already accommodates them via cross-identity parents.

## What this built
- The model realized in the data layer ([[kinora-5k13:task]]) and the kinograph format ([[kinora-zboo:spec]]).

## Consequences / follow-ups
- Fork resolution is branch-aware (HEAD's descendant wins) with a refuse-and-report fallback; bootstrap is detect-and-report only (see [[kinora-fhw1:spec-4]]).
- Current-name-drift warning for kinograph name hints deferred to a follow-up.
