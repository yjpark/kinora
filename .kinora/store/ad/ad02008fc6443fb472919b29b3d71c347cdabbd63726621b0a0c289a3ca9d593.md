# Task: CLI `render` command (mdbook output)

**Goal.** Resolve kinos and kinographs from the current branch's `.kinora/` and produce an mdbook site at the derived cache path, dispatching by kind and rewriting `kino://<id>/` URLs into cross-page links.

**Realizes:**
- [[kinora-zboo:task]] — kinograph rendering primitive it calls.
- [[kinora-6zxd:task]] — the resolver it walks.
- [[kinora-fhw1:spec-6]] — `kino://<id>/` link rewriting, kind dispatch.
- [[kinora-fhw1:spec-7]] — cache-path derivation.

**Done when:** `render` reads `.kinora/config.styx → repo-url` (errors if absent), derives the cache path per the normalization rules, renders the current branch end-to-end with markdown + kinograph kind dispatch, rewrites `kino://<id>/` URLs to cross-links, emits a `SUMMARY.md` organized by branch with source markers, does a full rebuild every run, and produces output viewable via `mdbook serve`. Verified by tests + an `mdbook build` smoke test.

## Outcome
Shipped library-first across four layers, 197 tests, zero warnings:
- **cache_path.rs** — pure `CachePath::from_repo_url()`: normalizes URLs (strip scheme/userinfo/`.git`, SSH `host:path` → `host/path`, lowercase host, preserve path case) → `<shorthash>-<name>`. Transport-agnostic (`https://` and `git@` collide); `strip_suffix(".git")` so `foo.git.git` ≠ `foo.git`.
- **render.rs (in-memory)** — `Book { pages, skipped }` via `render_for_branch(resolver, branch)`. Kind dispatch: markdown passthrough, kinograph via `Kinograph::render`, text fenced, binary placeholder, other → "unrenderable kind" placeholder. Forked identities (`MultipleHeads`) collected into `Book.skipped` instead of failing. `kino://<64hex>/` rewriting via `str::find` + `is_char_boundary` + `Hash::from_str` (UTF-8-safe). Slugs `<sanitized-name>-<shorthash>` with `-N` disambiguation.
- **render.rs (disk writer)** — `write_book()` wipes `src/`, emits `book.toml` (escaped title), `src/SUMMARY.md` grouped by branch, per-branch `index.md`, per-page files with a `*From branch <branch>*` footer. Per-branch dirs even in single-branch mode so multi-branch is additive.
- **CLI render.rs** — `kinora render [--cache-dir <path>]`; resolves cache root (XDG_CACHE_HOME → HOME/.cache), runs `render_for_branch` on the current lineage, writes under `<cache>/<shorthash>-<name>/`. Pure `resolve_cache_root(xdg, home)` factored out (workspace forbids `unsafe`, blocking `env::set_var`). Prints `rendered N pages [(skipped K forked)] into <path>`. `mdbook build` produced clean HTML.
