# `kinora clone` rebuilds `.kinora/` into a fresh directory

A rebuild primitive that re-runs committed content through the current write path
into a fresh directory — sanitizing legacy formats and dropping unreachable blobs
without hand-rolled migrations. Clone is also the atomic unit `kinora repack`
composes.

## The decision
- **Chosen: rebuild, not copy.** Enumerate content reachable from each root's heads, re-store each reachable blob through the current store API (filenames come out in current convention), rebuild root kinographs in current format, leave dst staged empty. Unreachable blobs in src are dropped — that is the cleanup payoff. Rejected a verbatim copy (would preserve legacy filenames and dead blobs).
- **Chosen: clone is hash-preserving — it rewrites filenames and drops unreachable blobs but does NOT re-serialize or migrate content.** Content migration (styx→styxl) is `kinora reformat`'s job. Two atomic operations kept separate.
- **Chosen: args are direct paths to `.kinora/` directories**, not enclosing repo roots — clone doesn't walk up and doesn't require being inside a repo. The `-C`/`--repo-root` flag is rejected when combined with clone (hard error).
- **Chosen: verify each src blob hashes to its claimed name on read; abort on mismatch.** Corruption is not something clone should silently paper over (verification is inherited from `ContentStore::read`).
- **Chosen: clone is not atomic — dst written incrementally, left partial on failure.** Acceptable because dst is a throwaway by convention; repack handles atomicity at the swap step. Author/provenance default to literal `"clone"` (no git derivation — may run outside a worktree).
- **Not a dead end:** future pack-file support slides in behind the same single "read blob by hash" / "write blob" store API the clone walk uses.

## What this built
- The `kinora::clone::clone_repo(src, dst, params) -> CloneReport` library module (reachability walk, re-store, root-kinograph rebuild, event filtering) plus a thin `kinora clone <src> <dst>` CLI. Verified by 13 library + 9 CLI tests (empty/single-kino/composition repos, unreachable-blob drop, legacy extensionless rewrite, hash preservation, error cases, summary formatting).

## Consequences / follow-ups
- Depends on the `--repo-root` flag bean [[kinora-jezf]] and on [[kinora-q6bo:spec]] (the commit lifecycle that moves history out of staged).
- **Spec deviation:** the bean's "staged must be empty" precondition was dropped — pre-[[kinora-bayr:task]] every repo has populated staged, so enforcing it would make clone impossible. Instead clone filters events during copy: store events with a reachable blob hash are kept, all assigns kept verbatim. The staged-empty check becomes natural once bayr/[[kinora-wcpp:task]] land.
- Composed into the atomic pipeline by [[kinora-0caz:spec]] (`kinora repack`).
