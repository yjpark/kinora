# Task: Render/sync engine — kinograph-bound, idempotent, region-preserving

**Goal.** Build `engine::sync_file`: given a file's `stencil:kinograph` binding,
load the api-kinograph, match each slot to an entry by name, resolve it (pin or
head), split the spec kino, and write the read-only block — preserving editable
regions byte-for-byte, idempotent on re-run, warning on hand-edited read-only
regions. TDD.

**Realizes:**
- [[kinora-q28s:spec]] — Spec kino content model: markdown prose + fenced code.
- [[kinora-thow:spec]] — Stencil marker protocol, region model, and LanguageTarget trait.

**Done when:** `sync_file` resolves the kind-checked api-kinograph, renders a
`///` doc + signature read-only block per slot indented to the slot, preserves
editable blocks byte-for-byte, classifies each slot
(Created/Updated/Unchanged/DriftOverwritten/Unmatched), reports unslotted
entries + orphan ro blocks, and a second run is a no-op; errors (NoBinding,
NotApiKinograph, NotApiSpec, DuplicateEntryName) are wired into `StencilError`;
all tests pass with zero warnings.

## Outcome
Added `engine::sync_file` — the render/sync core tying region + spec models to
kinora. Per-slot resolution (`pin` → `resolve_at_version`, else head), read-only
hash for idempotency, status classification, reports with
`changed()`/`drifted()`/`unmatched()`. Review hardening: tolerant per-entry
resolution (a broken unrelated entry no longer fails the file), loud
`DuplicateEntryName`. 67 tests; zero warnings. Unblocked the CLI beans
([[kinora-exay:task]], [[kinora-guv8:task]]).
