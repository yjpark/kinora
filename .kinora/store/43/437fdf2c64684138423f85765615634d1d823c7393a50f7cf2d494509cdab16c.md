# CLI packaging: two-crate workspace (kinora library + kinora-cli binary)

Settles RFC-0003 Q5 for the kinora bootstrap: how the code is packaged.

## The decision
- **Chosen: a two-crate workspace** — `kinora` (library: store, ledger, kinograph, resolve, render logic) and `kinora-cli` (thin binary wrapper, binary named `kinora` via `[[bin]] name`). Clean library boundary from day one; CLI-only deps don't pollute library consumers; matches the existing Cargo workspace.
- **Rejected: a single crate, and finer-grained module crates up front.** A single crate leaks CLI deps into library consumers; premature granular splitting adds churn. Internal modules (`store`, `ledger`, `kinograph`, `resolve`, `render`) live inside `kinora`.
- **Not a dead end:** finer-grained crate splits are a mechanical, non-breaking refactor for consumers and can happen later.
